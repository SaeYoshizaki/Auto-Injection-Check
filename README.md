# Auto Injection Check

## プロジェクト概要
このツールは、AIチャットボットに対して自動的に脱獄プロンプトを送信し、その応答を解析してセキュリティ脆弱性を診断するシステムです。Next.jsによるWeb管理画面から、ブラウザ自動操作を実行し、診断結果をPDFレポートとして出力します。

### 主な機能
* **Web UI操作**: ブラウザから簡単に設定・実行が可能。
* **自動ログイン**: 組織IDとユーザーID/PASSによる2段階認証に完全対応。
* **クリーンな検証**: チャットごとに「新規チャット」ボタンを自動クリックし、会話履歴をリセットして検証。
* **PDFレポート**: 診断結果を整形し、PDFで出力。

---

## 動作環境
* **Python**: 3.10 以上
* **Node.js**: 18.0 以上
* **Browser**: Chromium (Playwrightによって自動管理されます)

## 実行方法

### 1. バックエンドの起動
ターミナルを開き、以下の手順でPythonサーバーを起動してください。

```bash
cd ~/Desktop/auto-injection-check

source backend/venv/bin/activate

pip install fastapi uvicorn python-multipart fpdf2 playwright
playwright install

export REPORT_DATA_ROOT=~/Desktop/auto-injection-check
uvicorn backend.server:app --reload
```
### 2. フロントエンドの起動
新しいターミナルを開き、以下の手順でNext.jsを起動してください。

```bash
cd ~/Desktop/auto-injection-check/frontend

npm install

npm run dev
```


### 3. スキャンの実行
1.  ブラウザで **[http://localhost:3000](http://localhost:3000)** にアクセスします。
2.  以下の情報を入力します：
    * **Target URL**: チャットボットのURL
    * **Organization ID**: 組織コード
    * **Username / Password**: ログイン情報
    * **Scan Mode**: 診断モードを選択（後述）
3.  **"Start Security Scan"** ボタンをクリックします。


## スキャンモードについて
現在のバージョンでは、用途に応じて以下の4つのスキャンモードを選択できます。

| モード | テスト数 | 特徴 | 主な用途 |
| :--- | :---: | :--- | :--- |
| **test** | 3件 | 最小構成のテスト | 接続確認・動作確認 |
| **light** | 約20件 | 軽量スキャン | 開発中の簡易チェック |
| **standard** | 約60件 | 標準スキャン | 通常のセキュリティ診断 |
| **full** | 約200件以上 | フルスキャン | リリース前の詳細診断 |

各モードでは、攻撃プロンプトのカテゴリ配分や会話モードの割合、実行回数などがあらかじめ設定されています。
必要に応じて **詳細設定** から以下の項目をカスタマイズすることも可能です。

- 総テスト数
- ラウンド数（繰り返し回数）
- プロンプトバリエーション数
- 攻撃カテゴリの割合
- 会話モードの割合
- 実行順のランダム化
- 乱数シード（再現性確保用）

## プロンプトバリエーション

本ツールでは、プロンプトインジェクションの**表現の違いによる成功率の差**を評価するため、
1つの攻撃プロンプトに対して複数の表現バリエーションを生成してテストを行います。

現在サポートしている主なバリエーションは以下の通りです。

| バリエーション | 説明 |
| :--- | :--- |
| **polite** | 丁寧な依頼形式のプロンプト |
| **direct** | 強い命令形のプロンプト |
| **roleplay** | 権限や役割を与えるロールプレイ形式 |
| **indirect** | 間接的・婉曲的な表現 |
| **override** | 制約やポリシーを無視するよう誘導する表現 |
| **japanese_translation** | 日本語表現に変換した攻撃プロンプト |

これにより、以下の観点での脆弱性評価が可能になります。

- 表現の揺らぎに対する耐性
- 日本語 / 英語による攻撃成功率の違い
- 命令強度による成功率の差

これらの結果はスキャン結果に記録され、PDFレポートとして出力されます。
